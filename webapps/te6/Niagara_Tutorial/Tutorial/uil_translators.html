UIL Translators

The uil command allow you to add, remove, and manipulate translators.  The uil command is broken into the following sub-commands:
 
 
uil count
uil create
uil get autosense
uil get callback
uil get index
uil get version
uil info
uil list
uil load
uil message
uil name
uil parse
uil remove
uil set autosense
uil set callback
uil set index
uil set loglevel

Managing uil Translators

The command to see what uil translators are available is:
 
uil list
For example:

In this case, Niagara has three uil translators loaded.  Note the index, version, and number of devices connected.  To change the current uil, use the command uil set index (usi is a shortname alias). The format for this command is:
 

uil set index <index>
 
You can also create new UIL's and remove UIL's dynamically.  The command to create a new uil is:
 
uil create <driver name> ?option1? ?option2? ?...?
Here are some examples:
 
uil create test
uil create itech
uil create spti
uil create simscsi
uil create simscsi 1
Note the parameter we passed to the simscsi driver in the last example tells simscsi to start work.exe automatically.  Parameters after the uil name in the examples above are different depending on which driver being loaded (most drivers have no startup parameters).  Parameters are always optional, if not included the driver will load with a default configuration. In most cases, it is ok to have multiple instances of the same uil type.  To remove a uil translator, the command is:
uil remove <uil index>
 
If the simscsi driver automatically launched work.exe, it will automatically kill it when this command is called.
